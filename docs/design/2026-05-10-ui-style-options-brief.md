# EvoCraft UI Style Options Brief

创建日期：2026-05-10

## 目标

生成多套 EvoCraft 桌面端 UI 风格方向，供后续选择一个作为 MVP 的视觉基线。

## 共同约束

- 所有风格都必须保留 EvoCraft 是 AI 学习助手应用集合的定位。
- 所有风格都以 `错题收集` 作为第一版可用应用。
- 所有风格都必须体现 `原图` 与 `干净题面` 的区别。
- 所有风格都要避免营销落地页气质，第一屏应像真实可用的工具。
- 所有风格都要避免过度卡通、复杂吉祥物、强游戏化。

## 风格方向

### A. 清爽学习桌

关键词：安静、可信、亲和、轻教育工具。

适合：第一版 MVP，家长和孩子都容易接受。

视觉特征：

- 暖白背景。
- 柔和绿色和蓝色。
- 卡片轻量，信息层级清楚。
- 少量学习纸张和整理感隐喻。

### B. 未来学习工作台

关键词：AI、效率、结构化、现代桌面工具。

适合：强调 AI 能力和复核效率。

视觉特征：

- 更强的分栏布局。
- 蓝绿色功能色。
- 状态标签更清晰。
- 类似生产力工具，但保持儿童学习场景温度。

### C. 纸面笔记本

关键词：纸感、错题本、学习资料整理、温和。

适合：强调从真实作业纸面转成干净错题本。

视觉特征：

- 纸张纹理感但不能脏。
- 轻微手账感。
- 干净题面像整理后的错题卡。
- 控件仍保持现代 Web App 可用性。

### D. 成长收集册

关键词：轻游戏化、成长、收集、不过度娱乐。

适合：为后续金币、角色装扮、收藏体系留气质。

视觉特征：

- 柔和暖黄色作为奖励点缀。
- 学习应用入口有轻收藏感。
- 不能出现复杂游戏大厅或卡通角色。
- 仍以错题整理效率为中心。

## ImageGen Prompt

Use case: ui-mockup
Asset type: desktop app UI style exploration board
Primary request: Create a polished 16:9 UI style exploration board for EvoCraft, an AI learning assistant app collection for Shanghai children. Show four distinct visual style directions side by side in a 2x2 comparison board, each represented by a realistic desktop app mockup for the same wrong-question capture product flow.
Style directions:
A. 清爽学习桌: calm warm educational tool, warm white, soft green and blue, trustworthy, friendly, clean study desk feeling.
B. 未来学习工作台: modern AI productivity workspace, structured two-pane layout, blue-green accents, clear status chips, efficient but still warm.
C. 纸面笔记本: modern web app with subtle paper notebook feeling, clean worksheet cards, organized wrong-question notebook, light paper texture, not messy.
D. 成长收集册: restrained growth-and-collection feeling, soft warm yellow reward accents, future app collection hints, light gamification without mascot.
Shared product content: EvoCraft, AI 学习助手应用集合, 错题收集, 上传错题照片, 原图, 干净题面, 识别草稿, 去痕需复核, 保存错题.
Composition/framing: a clean design comparison board with four labeled quadrants A/B/C/D. Each quadrant shows one desktop screen mockup with App Hub or recognition review elements. Keep the UI readable enough to compare visual direction; labels can be concise.
Constraints: no dark theme, no marketing hero page, no cartoon mascot, no heavy gamification, no decorative blobs, no watermark, no lorem ipsum. The result should make the four style directions clearly different while preserving the same product requirements.
