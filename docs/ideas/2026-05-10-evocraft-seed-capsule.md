# EvoCraft 种子想法胶囊

创建日期：2026-05-10

## 胶囊记录铁律

任何新的 PRD、PRD 重要更新、明确产品想法，都必须同步进入这个想法胶囊。

记录方式：

- 按日期写入时间线。
- 提炼想法、原则、观点和产品判断。
- 不复制 PRD 原文。
- PRD 负责结构化需求，想法胶囊负责记录产品意图和演化脉络。

## 想法时间线

### 2026-05-10

- EvoCraft 的长期形态是 AI 学习助手应用集合，不是单一错题工具。
- 第一版 MVP 只让 `错题收集` 成为可用应用，但顶层需要保留 App Hub 和后续应用扩展空间。
- 错题收集的核心价值不是保存照片，而是把带有书写和批改痕迹的真实错题整理成可复习的干净题面。
- 原图和干净题面必须并存：原图用于溯源和复核，干净题面用于复习、分析和后续练习。
- 第一版采用本地优先和 AI provider-agnostic 思路；UI 阶段可以先 mock 识别结果，不应过早绑定具体模型服务。
- 儿童学习照片属于敏感材料，真实外部 AI/OCR 上传必须在隐私、授权、存储、删除边界明确后再做。
- 后续每次 PRD 或明确想法变化，都要把观点提炼回这个胶囊，并按时间线保留演化记录。
- 长期开发不能依赖聊天上下文；项目记忆、进度和想法演化必须写入仓库文档，并在进度记录里保留任务、完成情况、卡点、执行命令和下一步。
- 第一版 UI 原型应以桌面端工具感为主，使用 App Hub 展示应用集合身份，用四屏流程呈现上传、识别检查、去痕复核和保存结果。
- UI 风格选择先做横向探索：清爽学习桌、未来学习工作台、纸面笔记本、成长收集册四类方向都要服务同一个错题收集流程，不能为了风格牺牲可复核和可信任。
- 已选择 B 方案 `未来学习工作台` 作为第一版视觉基线：EvoCraft 应先像一个高效、可信的学习工作台，而不是普通拍题软件或游戏大厅。蓝色主色用于表达 AI 整理、结构化和可信复核，青绿色只作为辅助提示，清晰状态标签用于降低家长和孩子复核 AI 结果的成本。
- 正式 UI 图要为 coding 服务：优先表现可拆分的应用外壳、上传区、原图/干净题面、AI 复核面板、状态 chip 和保存详情，而不是只做风格氛围图。

## 原始想法

EvoCraft 是一个 AI 学习助手应用集合，第一批面向上海的小朋友。它不应该一开始就做成庞大的完整平台，而是先保留应用集合的顶层结构，再做出第一个最小但完整的应用闭环。

第一版 MVP 是 EvoCraft 应用集合里的第一个应用：`错题收集`。用户上传一张包含一道错题的照片，应用识别题目内容，支持语文、数学、英语，并且在图形题、公式、图表、几何图或识别不确定时保留视觉内容。因为真实错题通常带有学生书写、批改、圈画、涂改等痕迹，应用需要生成适合复习的干净题面，而不是直接把带作答痕迹的照片当成最终材料。

后续 EvoCraft 可以逐步扩展成更完整的学习助手应用集合：错题分析、复习、背单词、阅读辅助、游戏元素、金币收集、收藏循环、虚拟人物装扮等。

## 应用集合结构

EvoCraft 顶层需要保留 `App Hub / 应用集合首页` 的思路。第一版只让 `错题收集` 可用，但产品和 UI 不应把 EvoCraft 设计成只能做错题的单一工具。

未来应用可以包括：

- 背单词。
- 复习计划。
- 阅读辅助。
- 数学专项练习。
- 虚拟人物和装扮。
- 学习奖励与金币。

## 产品气质

EvoCraft 应该是“认真学习工具 + 轻度游戏化成长感”的结合，而不是普通拍题软件。名字里的 `Craft` 暗示孩子在逐步打造自己的学习资料、学习世界和成长路径。

## 第一个能力碎片

错题收集应用：

- 上传或拖入一张照片。
- 选择或自动判断科目。
- 从图片中识别一道题。
- 尽可能提取文字。
- 对图形、公式、几何图、表格等内容保留图片片段。
- 去除或隔离学生书写、批改、圈画等痕迹，生成干净题面。
- 用户可以修正识别结果。
- 保存一个干净的错题记录，默认展示去痕后的题面，原图保留用于溯源和重新处理。

## 长期能力碎片

- 按知识点和错误原因分析错题。
- 复习计划和练习循环。
- 生成相似题。
- 给家长或老师看的简要总结。
- 背单词应用。
- 金币和收藏物。
- 虚拟人物装扮。
- 孩子的学习进度地图。
- 桌面、平板、手机多端体验。

## 设计种子

- 桌面端优先，因为第一版需要仔细检查照片识别结果。
- 第一屏可以是轻量应用集合入口，明确当前可用应用是“错题收集”。
- UI 要让“不确定”可见：识别文本和图片片段可以并存。
- UI 要区分“原图”和“干净题面”，避免孩子复习时看到自己的错误作答痕迹。
- 不要让孩子面对太多复杂控制。
- 家长可以协助上传和修正，但保存后的错题记录应该让孩子也能看懂。
- 第一版最重要的是让一个错题记录显得完整、可信、可继续使用。

## 可升级为需求的条目

后续可以分别升级为 PRD 或设计文档：

- MVP PRD：EvoCraft 应用集合中的单道错题收集应用流程。
- UI 简报：App Hub、错题收集应用入口、桌面端屏幕结构和视觉方向。
- 识别规格：OCR、科目判断、图形保留、书写痕迹去除、置信度处理。
- 数据模型规格：错题记录字段和存储方式。
- 游戏化规格：金币、奖励、虚拟人物、复习循环。
