# MVP 收尾加固与技术路线设计

日期：2026-05-16

## 背景

错题收集原型已经具备上传、选择题目区域、候选框删除、手动画框、复核、保存、错题本和详情闭环。下一步目标不是继续堆功能，而是把原型变成可信的 MVP 基线，并快速完成技术路线决策。

## 设计结论

本轮收尾加固聚焦三个用户可感知风险：

1. 隐私边界：进入选题和识别前，用户必须确认当前 MVP 只做浏览器本地保存；未来外部 AI 上传需要单独授权。
2. 本地数据可控：错题本支持删除单条记录，详情页支持删除当前记录，错题本支持清空本地数据。
3. 失败可恢复：上传空文件、未确认隐私、候选框清空、自动找题失败、本地保存失败、本地删除/清空失败都要停留在可继续的页面并给出恢复动作。

技术路线采用保守推进：

- 当前继续静态 Web 和 `localStorage`。
- 下一阶段先抽 AI adapter 与 mock contract tests。
- React/Vite 作为复杂度触发后的迁移目标。
- Electron/Tauri 和后端延后到本地文件库、多端同步、账号或远端 AI 代理需求明确后。

## 交互边界

- 上传页：新增本地隐私确认，未确认时不能进入 `选择题目区域`。
- 选题页：没有选中区域时禁用确认识别，并在当前页提示手动画框或重新自动找题。
- 复核页：保存失败时停留复核页，避免用户以为记录已保存。
- 错题本：每条记录可删除，清空本地数据需要确认。
- 详情页：当前记录可删除，删除后返回错题本列表。

## 测试重点

- 纯函数测试覆盖记录删除后的选中态迁移、删除不存在记录时不污染原对象、本地保存成功/失败、本地清空成功。
- 静态结构测试覆盖隐私确认、区域错误、保存错误、删除记录和清空本地数据入口。
- 浏览器验收覆盖上传隐私 gate、保存、详情删除、错题本清空和空状态恢复。
